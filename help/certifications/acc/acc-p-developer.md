# Adobe 인증 여정 [!DNL Campaign Classic] Developer Professional

인증된 Adobe Developer Professional이 되는 방법을 알아봅니다.

## 최소 경험

다음을 수행해야 합니다.

* Adobe 1년 이상 [!DNL Campaign Classic] 소프트웨어 경험
* JavaScript 프로그래밍, ETL을 포함한 SQL 개발 및 웹 기술(XML, HTML 및 웹 서비스)에서 1년 이상의 통합 경험
* 마케팅 자동화에 대한 기본 이해
* 관계형 데이터베이스 개념 이해

>[!NOTE]
>
>합격하신 후 2년간 유효한 인증입니다 최신 제품 기능을 다루므로 가능한 한 최신 버전을 사용하는 것이 좋습니다.

* 기술 컨설턴트
* 주니어 마케팅 기술 엔지니어
* 주니어 캠페인 개발자

**시험 세부 정보:**

* 수준: Professional(0-12개월 경험)
* 점수 전달: 33/50
* 시간: 100분
* 배달: 온라인 처리(카메라 액세스 필요) 또는 테스트 센터 처리
* 사용 가능한 언어: 영어
* 비용: 125달러(글로벌) / 95달러(인도)
* 시험 ID: AD0-E331

**시험 목표 및 범위**

섹션 1: 시스템 구성(25%)

* 관리 노드에서 시스템 차원의 설정 구성을 보여 줍니다.
* 프로세스를 적용하여 OOTB 수신자 스키마를 확장합니다.
* 스키마 세부 정보 정의(기본 키, 자동 기본 키, 데이터베이스 인덱스)
* 적용 가능한 양식 렌더링 사용 구분
* 작업 공간/프로젝트 수준에 영향을 주는 시스템 전체 구성 설정 구분
* SFTP 외부 계정에 대한 구성 설정 식별
* 열기 및 닫힌 열거형에 대한 구성 설정을 식별합니다
* 유형화 규칙 유형 차별화
* 사용자 지정 Target 맵을 사용하기 위한 요구 사항 및 OOTB 기능에 미치는 영향 식별
* 다른 Adobe Experience Cloud 솔루션에서 외부에서 생성된 대상에 Adobe에서 액세스하는 방법을 설명합니다 [!DNL Campaign Classic]
* 상호 작용 및 오퍼에 대한 비즈니스 사용 사례 설명
* 메시지 센터에 대한 비즈니스 사용 사례 설명
* V8 API 데이터 가져오기 기능 식별

제2절: 사용자 지정(16%)

* 프로세스를 적용하여 Campaign 스키마 편집
* 설정을 식별하여 올바른 스키마 확장을 확인하거나 사용자 지정 스키마를 구성합니다
* 사용자 지정된 옵션에 대한 사용 사례 설명
* Seedlists의 사용 사례 설명
* OOTB 옵트아웃 / 웹 앱 구독 취소 수정
* 프로세스를 적용하여 컨텐츠 관리 구성 요소 만들기
* 콘텐츠 관리를 사용할 사용 사례 식별
* 프로세스에 적용을 통해 OOTB 양식 업데이트

섹션 3: 운영 보안(10%)

* 사용자 그룹 및 역할을 추가하는 프로세스를 설명합니다
* 사용자 권한 및 계층 구조를 올바르게 사용하여 사용자 관리 행렬을 변환하는 프로세스를 적용합니다
* 올바른 권한 식별 및 정의
* 사용자 그룹에 대한 보안 영역 식별

섹션 4: 기술 워크플로우(29%)

* 가져오기/내보내기에 프로세스 적용
* 워크플로우가 액세스할 수 있는 다양한 데이터 소스를 식별합니다
* 프로세스를 적용하여 데이터를 변환하고 작업 테이블을 조작합니다
* 적용 가능한 워크플로우 활동 및 해당 기능/속성을 열거합니다.
* 프로세스를 적용하여 워크플로우 속성 구성
* 표현식 편집기를 사용하여 활동 기능 확장
* OOTB 워크플로우와 사용자가 만든 워크플로우 간의 차이를 구별합니다
* OOTB 워크플로우의 목적 설명
* 사용자 지정 워크플로우 문제 해결
* 변수 유형과 범위 및 사용 구분
* OOTB 변수 및 해당 사용 식별
* 프로세스를 적용하여 경고 및 승인을 구성합니다
* 캠페인 데이터베이스 유형 식별

섹션 5: 인스턴스 유지 관리(4%)

* 다양한 서비스 로그에서 오류 식별
* 워크플로우 Heatmap 측정 설명

제6조: Campaign 구성(14%)

* 프로세스를 적용하여 캠페인 템플릿을 사용하여 캠페인 만들기
* 승인을 만드는 프로세스를 설명합니다.
* 감사 로그를 분석하여 오류 식별
* 캠페인 시나리오가 발생하면 다른 게재의 사용을 식별합니다
* 워크플로우에서 고급 프로세스를 만드는 프로세스를 정의합니다
* 워크플로우 실행 방법 설명
