# Adobe 인증 여정 [!DNL Campaign Classic] 개발자 전문가

인증된 Adobe Developer 전문가가 되는 방법을 알아봅니다.

## 최소 경험

다음을 수행해야 합니다.

* 최소 2년 동안 Campaign 소프트웨어 경험 제공
* JavaScript 프로그래밍, ETL을 포함한 SQL 개발 및 웹 기술(XML, HTML 및 웹 서비스)에서 3년 이상의 통합 경험
* 디지털 옴니채널을 사용하여 마케팅 캠페인을 실행하는 경험
* 데이터베이스 아키텍처 및 데이터베이스 마케팅에 대한 지식 경험
* 캠페인 기능에 요구 사항 매핑 기능
* Campaign 구현 문제 해결 기능
* Campaign과 관련된 서버 구성에 대한 기본 이해

>[!NOTE]
>
>합격하신 후 2년간 유효한 인증입니다 최신 제품 기능을 다루므로 가능한 한 최신 버전을 사용하는 것이 좋습니다.

**시험 세부 사항**

* 수준: 전문가(1-3년 경험)
* 점수 전달: 34/50
* 시간: 100분
* 배달: 온라인 처리(카메라 액세스 필요) 또는 테스트 센터 처리
* 사용 가능한 언어: 영어
* 비용: 225달러(글로벌) / 150달러(인도)
* 시험 ID: AD0-E330

**준비 자체 평가**

을(를) 완료합니다 [온라인 준비 질문](https://scorpion.caveon.com/launchpad/ad-q-e129-readiness-questionnaire-for-adobe-aem-assets-developer-professional-exam-copy-21ocv4/ad-q-e312-readiness-questionnaire-adobe-campaign-classic-developer-expert-exam){target="_blank"} 현재 경험이 권장되는 최소 후보 자격과 일치하는지 확인하기 위해 다음을 수행하십시오.

**시험 목표 및 범위**

섹션 1: 시스템 구성(16%)

* 시스템 전체 구성 설정 및 기본값 적용
* 다양한 매개 변수가 제공되면 적절한 Adobe Campaign 기능을 선택하여 사용 사례를 구현합니다
* 시스템 구성 설정 문제 해결
* 사용자 지정 구성 구현
* Campaign Campaign 컨트롤 패널 구성 기능 식별

제2절: 사용자 지정(22%)

* 요구 사항 세트가 주어지면 모범 사례를 적용하여 데이터 모델을 디자인합니다
* Campaign에서 API 구현 설명
* Adobe Campaign에서 JavaScript를 사용한 개발에 대한 우수 사례 설명
* 워크플로우, SOAP 호출, 유형화 규칙 및 개인화 블록에서 JavaScript 라이브러리 적용
* Adobe에서 SQL 함수 사용에 대한 모범 사례 설명 [!DNL Campaign Classic]
* 시나리오가 주어지면 Adobe에서 SQL 스크립트를 사용하는 방법을 이해합니다 [!DNL Campaign Classic]
* 일반적으로 사용되는 OOTB 함수 설명

섹션 3: 액세스 관리(8%)

* 시나리오가 발생하면 연산자에 대한 보안 설정을 구성합니다
* 솔루션 디자인 문서를 기반으로 액세스 관리 설정을 구성하는 방법을 설명합니다
* Campaign 컨트롤 패널을 사용하여 액세스 관리 설명
* 데이터 파티셔닝의 제한 사항 및 sysfilter 사용 시기 파악

섹션 4: 데이터 통합(16%)

* 시나리오가 있다면 외부 데이터 소스와 통합할 최상의 방법을 선택하십시오
* 워크플로우 데이터 처리에 대한 모범 사례 구현
* 워크플로우 내에서 데이터 통합 기법 적용
* 외부 시스템에서 Adobe Campaign과 통합할 보안 및 네트워크 설정을 식별합니다
* Adobe Campaign 외부에서 스키마에 대한 CRUD 프로세스를 설정합니다

섹션 5: 인스턴스 유지 관리(12%)

* 워크플로우 Heatmap을 이해하고 그에 따라 행동합니다
* 인스턴스 모니터링을 위해 사용자 지정 워크플로우 또는 보고서를 만드는 프로세스를 보여줍니다
* 시스템 표시기를 기반으로 문제 식별
* 서비스 및 프로세스가 시스템에 미치는 영향 설명
* 기본 시스템 감사 지표/검사 표시

제6조: Campaign 구성(25%)

* 고급 개인화 기법 적용
* 사용자 지정 대상 매핑 구성 및 적용
* Campaign 워크플로우에서 적절한 JavaScript 사용을 식별하고 그 이유를 확인합니다
* 사용자 지정 코드를 적용하여 임시 테이블 및 워크플로우 컨텍스트 조작
* 워크플로우 로직의 유효성 검사 및 디버그
* 고급 게재 설정 적용
* 성능에 부정적인 영향을 주는 워크플로우 사례 설명
* 사용 사례가 있다면 복잡한 타겟팅 워크플로우 디자인을 추천합니다
