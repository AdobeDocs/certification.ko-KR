# 에 대한 인증 여정 [!DNL Adobe Analytics] Developer Professional

인증을 받는 방법 알아보기 [!DNL Analytics] 개발자 전문가.

## 최소 경험

다음을 수행해야 합니다.

* 6-12개월 경험 [!DNL Adobe Analytics] 구현(웹 및 모바일 앱)
* 비즈니스 요구 사항이 SDR(기술 솔루션)으로 변환되는 방법과 애널리스트가 제공한 요구 사항을 기반으로 이해하려면 IT와 비즈니스 간 브리징에서 Analytics 구현 팀을 지원하는 플레이어 역할을 합니다
* 필요한 변수 유형, 데이터 수집 위치 및 방법, 데이터 모델(데이터 계층)을 디자인, 태그 관리 솔루션을 구성 및 유지 관리하는 방법을 이해합니다

다음 작업이 있는 경험:

* SDR(솔루션 디자인 문서) 해석 및 모든 목적 이해 [!DNL Adobe Analytics] 수집된 변수
* Adobe(Experience Platform) Launch 이해 및 작업
* 기준 요소 구성 이해 [!DNL Adobe Analytics] 차원 및 이벤트 페이지 이름, 링크 이름, Activity Map 등을 사용할 수 있지만 전문 개발자와 협력하여 더 많은 고객 구현을 위해 사용할 것입니다. 변수 매핑은 여전히 전문가 수준 개발자가 소유합니다
* AA 전문가가 제공한 디자인에 따라 Launch 내에서 규칙 만들기 및 구성
* charles logs와 같은 웹 콘솔 디버거 또는 모바일 앱 시스템을 활용하여 변수 및 이벤트 실행에서 캡처되는 데이터를 테스트하고 확인합니다
* 데이터 계층의 정의 및 JavaScript 개체를 읽는 방법을 알고 있습니다
* 보고서, 세그먼트 및 액세스 요청을 위한 프로세스를 공유하지만 사용자 그룹, 거버넌스 프로토콜을 관리하고 SDR을 관리하는 방법은 아닙니다.
* 지침 제공 및 모바일/SDK 솔루션 구성
* 비디오 추적(이벤트 기반)에 대한 기술 자료 설명서에 대한 일반 프레임워크 이해
* Analysis Workspace에서 기본 보고를 수행하고 다음 위치에서 사용할 수 있는 데이터 출력을 이해합니다 [!DNL Adobe Analytics] Data Warehouse
* Adobe Experience Platform Launch을 통한 기본 경험(Adobe Launch)

>[!NOTE]
>
>합격하신 후 2년간 유효한 인증입니다 최신 제품 기능을 다루므로 가능한 한 최신 버전을 사용하는 것이 좋습니다.

**시험 목표 및 범위**

섹션 1: Adobe Experience Cloud 에코시스템의 Analytics(14%)

* Adobe Experience Cloud ID의 기능 확인
* Adobe Launch에 대한 사용 식별
* 활성화/지원 방법 확인 [!DNL Adobe Analytics] Adobe Experience Cloud 에코시스템의 태그

제2절: 솔루션 디자인 참조를 기반으로 한 분석 전략 및 디자인(12%)

* 솔루션 디자인 참조가 주어지면, 채워야 하는 데이터 개체를 식별합니다
* 기술 사양 문서의 일부가 주어지면 데이터 캡처 구현 방법을 확인합니다

섹션 3: Analytics 구현 및 구성(30%)

* 배포할 단계 식별 [!DNL Adobe Analytics] 코드
* Analytics 추적을 위한 최소 배포 코드의 함수 식별
* 내에서 다양한 구성 요소를 구성하는 단계 식별 [!DNL Adobe Analytics] UI
* 의 함수 식별 [!DNL Adobe Analytics] 보고서 세트 및 변수 설정
* 데이터 레이어 및 데이터 계층 값의 특성 식별
* 링크 추적 및 전자 상거래 추적과 같은 추적 기능을 위한 앱 측정 함수 식별

섹션 4: 태그 관리 시스템(18%)

* 태그 감사를 사용하여 요구 사항이 충족되었는지 확인합니다
* Launch를 사용하여 웹 사이트 태깅을 구성하는 단계 식별
* 시나리오가 제공되면 Launch에서 확장을 활성화, 수정 및 해결하는 방법을 식별합니다
* 규칙 및 데이터 요소를 만드는 방법 식별

섹션 5: Mobile Services 및 API(8%)

* 의 함수 식별 [!DNL Adobe Analytics] API(데이터 피드, 웨어하우스, 데이터 소스, 보고서)
* 처리 규칙 및 모바일 서비스의 특성 식별

제6조: 테스트, 유효성 검사 및 문제 해결(18%)

* 일반적인 JavaScript 오류의 의미를 식별합니다
* 공통 의미를 식별합니다 [!DNL Adobe Analytics] 변수 오류
* 시나리오가 주어지면 따라야 하는 테스트 및 디버깅 유효성 검사 우수 사례를 식별합니다
* 추적 방법 식별 [!DNL Adobe Analytics] 라이프사이클에서 서버 호출(예: 비콘 요청에서 보고서에 표시까지)
