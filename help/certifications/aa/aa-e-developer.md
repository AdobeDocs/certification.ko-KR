# 에 대한 인증 여정 [!DNL Adobe Analytics] 개발자 전문가

인증을 받는 방법 알아보기 [!DNL Adobe Analytics] 개발자 전문가.

## 최소 경험

2년 정도 경력을 가진 하이브리드 역할에서 구현 컨설턴트로 경험이 있어야 합니다 [!DNL Adobe Analytics] 구현 을 참조하십시오. IT와 비즈니스 간의 브리지 역할을 하고, 비즈니스 요구 사항을 SDR(기술 솔루션)로 변환하며, 분석가가 제공하는 요구 사항에 따라 데이터를 수집하는 데 필요한 변수 유형, 데이터 모델(데이터 계층)을 디자인하고, 태그 관리 솔루션을 구성 및 유지 관리하는 방법을 이해합니다.

Analytics 개발자는 거의 페이지에 태그를 구현하지 않습니다. 대신 IT 또는 시스템 통합자에 의해 구현됩니다.

다음 도구 및 환경에 익숙해야 합니다.

도구:

* Adobe Experience Platform Launch(Adobe 시작)
* 관련 디버깅 도구 [!DNL Adobe Analytics] (Adobe 디버거, 브라우저 개발자 도구, Charles와 같은 패킷 모니터)
* [!DNL Adobe Analytics] 관리 콘솔 보고서 세트 설정
* 기본 작업 공간 지식
* 웹 개발 도구, 코드 편집기 및 모바일 디버깅 도구
* Adobe Auditor
* 처리 규칙
* 분류 규칙 - 정규 분류 규칙
* JavaScript 오류
* 사용 가능한 컨텐츠에 대한 기본 이해 [!DNL Adobe Analytics] 관련 API
* Adobe I/O 통합

환경:

* 프런트엔드 웹 개발 환경
* 모바일 애플리케이션에 대한 지식(예: 제한 사항 및 차이점)을 참조하십시오
* 디지털 마케팅 분야
* PC, Mac, Android/iOS 운영 체제를 비롯한 백엔드 개발 환경에 대한 최소한의 지식(개발용이 아닌 기본 작업)

>[!NOTE]
>
>합격하신 후 2년간 유효한 인증입니다 최신 제품 기능을 다루므로 가능한 한 최신 버전을 사용하는 것이 좋습니다.

* 구성 [!DNL Adobe Analytics] 보고서 세트 및 변수 설정
* 배포 내에서 데이터 레이어 사용
* 링크 추적 및 제품 문자열과 같은 고급 추적 기능 실행

섹션 4: 태그 관리 시스템(20%)

* 에 Adobe Experience Platform Launch을 배포하기 위한 요구 사항을 열거합니다. [!DNL Adobe Analytics]
* 프로세스를 적용하여 Adobe Launch를 사용하여 웹 사이트 태깅 구성
* 확장 및 Adobe Launch를 활성화하기 위한 요구 사항을 열거합니다.
* 규칙 및 데이터 요소 만들기

섹션 5: Mobile Services(8%)

* 프로세스를 적용하여 모바일에 대한 기본 사전 배포 코드를 구성하고 실행합니다.
* 처리 규칙 및 모바일 서비스에 대한 이해 보기

제6조: 테스트, 유효성 검사 및 문제 해결(20%)

* JavaScript 오류 문제 해결
* 문제 해결 [!DNL Adobe Analytics] 변수
