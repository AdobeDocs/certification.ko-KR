## 시험 설명 {#exam-description}

{{exam-description}}

### 시험 세부 정보 {#exam-details}

* 레벨: 전문가(0~12개월 경력)
* 합격 점수: 33/51
* 시간: 102분
* 게재: 온라인 처리됨(카메라 액세스 필요)
* 사용 가능한 언어: 영어
* 비용: 125달러(전세계) / 95달러(인도)
* 시험 ID: AD0-E902

<br>

### 의도한 대상 {#intended-audience}

* 개발자, 자동화 엔지니어
* Automator, 프로세스 최적화 및 자동화 관리자
* 통합 컨설턴트, 컨설턴트, 파트너
* 시스템 통합자/파트너
* 비즈니스 분석가, 응용 비즈니스 분석가
* 시스템 관리자(IT, MIS, 기술, 지식 시스템, AI 관리자)

<br>

### 최소 경험 {#minimum-experience}

다음을 수행해야 합니다.

* [!DNL Workfront] API 및 API 탐색기를 사용할 수 있지만 API 호출에 능숙할 필요는 없습니다.
* [!DNL Workfront] 및 기타 API로 작업하기 위해 사용자 지정 API 호출을 사용하는 기능
* [!DNL Workfront] 앱에 대한 모듈 및 만들기, 읽기, 업데이트, 삭제 및 검색과 같은 기타 작업을 사용하는 기능
* [!DNL Fusion] 가드레일에 도달하지 않고 비즈니스 결과를 생성하는 실행 가능한 솔루션을 구축할 수 있습니다.
* 비즈니스 결과를 달성하기 위해 적절한 트리거 모듈을 선택하는 기능
* 흐름 제어 사용 방법에 대한 이해(경로, 반복자/집계자, 흐름 설명)
* 기본적인 문제 해결 및 오류 처리를 수행하는 기능
* 시나리오를 효과적으로 테스트하고 실행할 수 있는 기능
* 조직 팀 및 사용자를 관리하는 기능
* REST API Oauth2용 범용 커넥터의 목적 및 사용에 대한 이해

<br>

### 시험 목표 및 범위 {#exam-objectives}

{{exam-objectives}}

**섹션 1: 기본 기술 개념(39%)**

* 다른 필드 형식이어야 하는 필드 형식이 하나인 [!DNL Fusion] 시나리오가 주어지면 데이터를 올바르게 변환하는 데 사용할 수 있는 함수를 제공합니다
* 함수를 다른 함수 내에 중첩해야 하는 [!DNL Fusion] 시나리오가 주어지면 올바른 형식의 식을 선택하십시오
* 팀을 사용하여 연결 및 기타 [!DNL Fusion] 기능에 대한 액세스를 제어하는 방법 식별
* [!DNL Fusion] 시나리오가 주어지면 시스템 및 기능 제한 사항을 확인하십시오.
* 시간대 조작이 필요한 [!DNL Fusion] 시나리오가 주어지면 사용자, 표시되는 조직 및 시간대 재정의가 필요한 위치를 구별합니다
* [!DNL Fusion] 시나리오가 주어지면 적절한 [!DNL Workfront] 모듈 또는 작업을 선택하십시오.
* [!DNL Fusion] 시나리오가 주어지면 입력 데이터를 출력 데이터로 적절하게 변환하는 매핑 패널 식 및/또는 모듈을 선택합니다
* [!DNL Fusion] 개발 도구를 사용하여 실행 오류를 해결하거나 서드파티 시스템에 대해 수행된 호출 및 응답을 확인하는 올바른 방법을 지정하십시오.
* 한 경로의 데이터가 다른 경로에 필요한 [!DNL Fusion] 시나리오가 주어지면 추가 라우팅 경로에서 해당 데이터를 사용할 수 있도록 하는 set-get 솔루션을 제공합니다
* 비즈니스 요구 사항에 맞게 [!DNL Fusion] 템플릿을 사용하거나 만드는 방법 식별
* [!DNL Fusion] 시나리오가 주어지면 실행 보기 및/또는 오류 해결을 위해 수행할 작업을 확인하십시오.
* 시나리오 요구 사항을 해결하기 위해 [!DNL Fusion] 시나리오 설정에서 다른 옵션을 식별합니다
* [!DNL Fusion] 시나리오가 주어지면 적절한 흐름 제어를 선택하거나 흐름 제어 모듈의 올바른 조합을 결정하십시오

**섹션 2: 시나리오 디자인 및 아키텍처(35%)**

* 제공된 단계 목록에서 JSON을 구문 분석하고 번들로 변환할 올바른 시퀀스를 선택합니다
* 요구 사항 세트가 주어지면 올바른 데이터 조회 방법을 식별합니다
* 인스턴트 트리거와 예약된 트리거 간의 차이점 확인
* 문서를 [!DNL Workfront]에 업로드하기 위한 올바른 모듈을 확인하십시오.
* 비즈니스 요구 사항이 있는 경우 [!DNL Fusion]이(가) 레코드를 처리했는지(사용자가 아님) 및 언제 처리했는지 추적하는 올바른 방법을 지정하십시오.
* [!DNL Fusion] 시나리오가 주어지면 디자인을 단순화하거나 유지 관리를 위해 최적화할 수 있는 기회를 식별합니다
* 시나리오 문서화를 위한 [!DNL Fusion] 모범 사례 식별
* 상황이 주어지면 시나리오를 통해 데이터 흐름을 줄일 수 있는 기회를 식별합니다
* 검색 모듈 및 다음 집계기에 표시되는 번들 수의 예를 고려할 때 번들 관리자에서 처리 및 표시하는 작업 수의 변경 사항을 설명합니다
* 번들 관리자에 있는 정보 식별
* 60일 이상 시나리오를 보관하고 가능한 유일한 방법(블루프린트 가져오기/내보내기)을 선택해야 하는 필요에 대한 설명이 제공됨
* 오류 403의 예: 금지됨, 오류 메시지의 출처를 올바르게 선택 및 원인

**섹션 3: 테스트 및 오류 처리(16%)**

* 테스트 계획의 요소 식별
* 테스트 사례의 요소 식별
* [!DNL Fusion] 시나리오가 주어지면 신뢰할 수 없는 서비스를 처리할 적절한 지시문을 올바르게 식별합니다
* [!DNL Fusion] 시나리오에서 사용자 지정 오류 처리를 추가하는 프로세스를 식별합니다
* 불완전한 실행을 추적, 읽기 및 해결하는 방법 식별
* 필수 데이터가 누락된 [!DNL Fusion] 시나리오가 주어지면 잘못된 데이터를 처리할 방법을 선택하십시오.

**섹션 4: API 작업(10%)**

* 429: 너무 많은 요청을 반환하는 서드파티 API가 주어지면 오류를 방지하거나 처리할 솔루션을 식별합니다
* [!DNL Workfront] 모듈에서 새 기능을 사용할 수 없는 [!DNL Fusion] 시나리오가 주어지면 새 기능을 사용할 올바른 참조 및 모듈 유형을 식별하십시오
* 전용 앱이 없는 서드파티 시스템이 제공되면 HTTP 앱을 식별하고 적절한 모듈을 선택합니다
* CRUD 작업과 REST API의 다른 공통 기능의 차이점 확인
