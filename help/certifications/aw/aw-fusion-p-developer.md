# Adobe 인증 여정 [!DNL Workfront Fusion] Developer Professional

인증된 Adobe Developer Professional이 되는 방법을 알아봅니다.

## 최소 경험

다음을 수행해야 합니다.

* 사용 능력 [!DNL Workfront] API 및 API 탐색기이지만 API 호출에 반드시 능숙하지는 않습니다
* 작업을 위해 사용자 지정 API 호출을 사용하는 기능 [!DNL Workfront] 및 기타 API
* 에 모듈을 사용할 수 있는 기능 [!DNL Workfront] 앱 및 만들기, 읽기, 업데이트, 삭제 및 검색과 같은 기타 작업
* 비즈니스 성과를 달성하지 않고도 얻을 수 있는 솔루션을 구축할 수 있는 능력 [!DNL Fusion] 가드 레일
* 비즈니스 결과를 달성하기 위한 적절한 트리거 모듈을 선택할 수 있는 기능
* 흐름 제어 사용 방법에 대한 이해(경로, 반복/누적, 흐름 설명)
* 기본적인 문제 해결 및 오류 처리를 수행하는 기능
* 시나리오를 효과적으로 테스트하고 라이브로 전환하는 기능
* 조직 팀 및 사용자를 관리하는 기능
* REST API Oauth2용 Universal Connectors의 목적 및 사용에 대한 이해

>[!NOTE]
>
>합격하신 후 2년간 유효한 인증입니다 최신 제품 기능을 다루므로 가능한 한 최신 버전을 사용하는 것이 좋습니다.

**시험 목표 및 범위**

섹션 1: 기본 기술 개념(39%)

* 주어진 [!DNL Fusion] 한 필드 형식이 다른 필드 형식을 사용해야 하는 시나리오는 데이터를 올바로 변환하는 데 사용할 수 있는 함수를 제공합니다
* 주어진 [!DNL Fusion] 함수가 다른 함수 내에 중첩되어야 하는 시나리오가 올바르게 형성된 표현식을 선택합니다
* 팀을 사용하여 연결 및 기타 항목에 대한 액세스를 제어하는 방법 확인 [!DNL Fusion] 기능
* 주어진 [!DNL Fusion] 시나리오, 시스템 및 기능 제한 사항 식별
* 주어진 [!DNL Fusion] 시간대 조작이 필요한 시나리오, 사용자, 표시 가능한 조직, 시간대 재정의가 필요한 위치를 구별합니다.
* 주어진 [!DNL Fusion] 시나리오에서 적절한 [!DNL Workfront] 모듈 또는 작업
* 주어진 [!DNL Fusion] 시나리오에서 입력 데이터를 출력 데이터로 적절하게 변형하는 매핑 패널 표현식 및/또는 모듈을 선택합니다
* 를 활용할 올바른 방법을 확인합니다. [!DNL Fusion] 개발 도구를 사용하여 실행 오류를 해결하거나 타사 시스템에 대한 호출 및 응답을 결정할 수 있습니다
* 주어진 [!DNL Fusion] 한 경로의 데이터가 다른 경로에 필요한 시나리오에서 이 데이터를 추가 라우팅 경로에서 사용할 수 있도록 하는 set-get 솔루션을 제공합니다
* 를 사용하거나 생성하는 방법을 식별합니다 [!DNL Fusion] 비즈니스 요구 사항 템플릿
* 주어진 [!DNL Fusion] 시나리오, 실행 보기 및/또는 오류 해결 방법 확인
* 에서 다양한 옵션을 식별합니다 [!DNL Fusion] 시나리오 요구 사항을 해결하기 위한 시나리오 설정
* 주어진 [!DNL Fusion] 시나리오에서 적절한 흐름 제어를 선택하거나 흐름 제어 모듈의 올바른 조합을 결정합니다

제2절: 시나리오 디자인 및 아키텍처(35%)

* 단계 목록이 주어지면 JSON을 구문 분석하고 번들로 변환할 올바른 시퀀스를 선택하십시오
* 요구 사항 세트가 주어지면 올바른 데이터 조회 방법을 식별합니다
* 즉시 트리거와 예약된 트리거 간의 차이점을 식별합니다
* 에 문서를 업로드하기 위한 올바른 모듈 식별 [!DNL Workfront]
* 비즈니스 요구 사항이 주어지면 이를 추적하기 위한 올바른 접근 방식을 파악하십시오 [!DNL Fusion] 사용자가 아닌 레코드를 처리했으며
* 주어진 [!DNL Fusion] 시나리오, 설계 단순화 및/또는 유지 관리를 위한 최적화 기회를 식별합니다
* 식별 [!DNL Fusion] 시나리오 문서화 우수 사례
* 상황에 따라 시나리오를 통해 데이터 흐름을 줄일 수 있는 기회를 식별합니다
* 검색 모듈 및 다음 집계에 표시되는 번들 수의 예를 보면 번들 관리자가 처리하고 표시하는 작업 수의 변경 사항을 설명합니다
* 번들 검사기에 있는 정보 식별
* 60일 이상 시나리오를 보관하고 유일한 방법(가져오기/내보내기 블루프린트)을 선택해야 하는 상황에 대한 설명이 제공됩니다
* 오류 403의 예를 들면 다음과 같습니다. 오류 메시지의 원본 및 원인을 올바르게 선택할 수 없습니다.

섹션 3: 테스트 및 오류 처리(16%)

* 테스트 계획의 요소 식별
* 테스트 사례 요소 식별
* 주어진 [!DNL Fusion] 시나리오, 신뢰할 수 없는 서비스를 처리하는 적절한 지시문을 올바르게 확인합니다.
* 에서 사용자 지정 오류 처리를 추가하는 프로세스를 확인합니다. [!DNL Fusion] 시나리오
* 불완전한 실행을 추적, 읽기 및 해결하는 방법을 확인합니다
* 주어진 [!DNL Fusion] 필요한 데이터가 없는 시나리오에서 잘못된 데이터를 처리하는 방법을 선택하십시오

섹션 4: API 작업(10%)

* 429를 반환하는 타사 API가 주어집니다. 너무 많은 요청이 있으면 오류를 방지하거나 처리할 솔루션을 식별합니다
* 주어진 [!DNL Fusion] 시나리오에서 새 기능을 사용할 수 없는 경우 [!DNL Workfront] 모듈, 새 기능을 사용할 올바른 참조 및 모듈 유형을 식별합니다
* 전용 앱이 없는 타사 시스템이 제공되는 경우 HTTP 앱을 식별하고 해당 모듈을 선택합니다
* CRUD 작업과 REST API의 기타 일반적인 기능 간의 차이점을 식별합니다
