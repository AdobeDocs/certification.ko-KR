# Adobe을 위한 인증 여정 [!DNL Real-Time Customer Data Platform] 개발자 전문가

{{intro}}

## 시험 설명 {#exam-description}

{{exam-description}}

<br>

### 의도한 대상 {#intended-audience}

* 소프트웨어 개발자
* 데이터 엔지니어
* 데이터 설계자
* 솔루션 설계자
* 엔터프라이즈 아키텍트

<br>

### 최소 경험 {#minimum-experience}

다음을 수행해야 합니다.

* 세그먼트 평가 차이 이해(스트림, 일괄 처리, 에지)
* API 사용 사례 이해(개발자 프로젝트 빌드 방법, 인증 작동 방법)
* ID 서비스 이해
* 데이터 흐름 이해
* Adobe I/O 이해
* 관계 작동 방식 이해
* 고객 사용 사례를 위한 데이터 모델(스키마, ID, 관계) 구축 기능
* 병합 정책을 기반으로 데이터 병합 전략 수립
* 데이터 레이블 지정, 정책 적용 및 Privacy Service의 올바른 방법을 결정하는 방법 이해
* 데이터 수집 패턴 제한에 대한 숙지
* 대상 및 사용 방법 이해
* Edge 데이터 수집 프레임워크 이해
* 구성해야 하는 AEP 권한(즉, 속성 수준 권한)을 결정하는 기능(클라이언트에 대한 권장 사항 제공)
* 쿼리 서비스 사용 사례에 대한 기본 이해
* 소스/대상 SDK에 대한 기본 이해

다음 개념에 익숙해야 합니다.

* 엔티티 관계형 다이어그램
* 관리자/권한
* Experience Platform API
* Adobe Experience Platform Debugger
* 웹 및 모바일 SDK

다음과 같은 환경에서는 편안해야 합니다.

* AEP
* 에지 네트워크

<br>

### 시험 세부 정보 {#exam-details}

* 레벨: 전문가(1-3년 경력)
* 합격 점수: 45/68
* 시간: 136분
* 게재: 온라인 처리됨(카메라 액세스 필요)
* 언어: 영어
* 비용: $225(글로벌) / $150(인도)
* 시험 ID: AD0-E605

<br>

### 시험 목표 및 범위 {#exam-objectives}

{{exam-objectives}}

**데이터 아키텍처 19%**

* 시나리오를 기반으로 RDBMS를 Adobe으로 번역 [!DNL RT-CDP]의 NoSQL 데이터 모델
* 실시간 고객 프로필 모델링을 위한 모범 사례 시연
* 모델링을 위한 ID 전략 및 관계 설계

**실시간 고객 프로필 15%**

* 프로필 어셈블리 작동 방식 설명
* ID 그래프의 작동 방식 설명
* 프로필 구성 설명
* Edge와 Hub 프로필 간 차이점

**데이터 수집 13%**

* CDP를 통한 데이터 수집 기능 설명
* Edge 수집의 기능 설명
* 데이터 수집 아키텍처에 대한 고급 기법 설명

**세그먼테이션 18%**

* CDP 내에서 대상 및 세그먼트를 작성하는 다양한 방법 설명
* 다양한 세분화 유형 및 작동 방식 설명
* 세그먼트/대상 활성화에 사용 사례를 적용하는 방법에 대한 이해 설명

**활성화 12%**

* CDP 내의 다양한 활성화 패턴 설명
* 활성화 관련 보호 기능 설명
* 사이트 개인화에 대해 설명

**거버넌스**

* Adobe에서 거버넌스에 대한 모범 사례 적용 [!DNL RT-CDP] 구현
* 다양한 서비스에서 데이터를 관리하는 방법 설명

**투여 15%**

* 모니터링 기능 이해
* 경고 설명
* 다양한 라이선스 사용 설명
* Adobe 내 ABAC 활성화를 위한 사용 사례 설명 [!DNL RT-CDP]
