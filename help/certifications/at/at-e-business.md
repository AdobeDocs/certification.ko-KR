# 에 대한 인증 여정 [!DNL Adobe Target] 비즈니스 전문가

공인 Adobe 비즈니스 전문가가 되는 방법을 알아봅니다.

## 최소 경험

다음을 수행해야 합니다.

* 다음을 포함한 최적화 플랫폼을 사용한 2년 이상의 경험 [!DNL Adobe Target]
* 사용 경험 [!DNL Adobe Target]
* 최적화 프로그램을 주도하거나 참여하기 위한 2년 이상의 경험
* Target/Adobe Marketing Cloud 관리 작업(환경 설정, 사용자 역할 및 권한, 설정)의 기본 사항에 대한 이해
* 비즈니스 KPI를 파악하고 최적화 목표로 변환하는 기능
* 요구 평가를 기반으로 제안 기능
* 적절한 [!DNL Adobe Target] 최적화 목표를 기반으로 한 활동 유형
* HTML 및 CSS를 사용하여 사용자 경험을 최적화하는 방법에 대한 지식
* 개념으로서 JavaScript 이해
* 최적화에 적용되는 통계 개념 및 데이터 분석에 대한 지식
* 3단계 워크플로우(모든 보고 소스를 포함하는 경험, 타깃팅, 목표 및 설정)에 대한 심층적인 지식입니다
* Activity 보고 결과를 실행 가능한 통찰력 및 제안으로 변환하는 기능
* 기술 변경이 Target(ITP, GDPR/CCPA, SPA)에 미치는 영향에 대한 이해
* 권장 사항 활동을 솔루션으로 사용하는 방법에 대한 이해

또한 다음 기술 및 환경에 익숙해야 합니다.

기술:

* Marketing Cloud 디버거 / Adobe Experience Cloud 디버거라고도 함
* Adobe Analytics을 Target용 보고 소스로서, 특히 A4T 통합
* 브라우저의 콘솔/브라우저 개발자 도구(예: 콘솔, 네트워크 탭)
* 브라우저의 네트워크 도구
* 다른 Adobe 도구에 대한 기본 이해
* 개발자 콘솔(모든 웹 브라우저)
* [!DNL Adobe Target] 샘플 계산기
* Adobe Audience Manager 기본 사항
* Adobe Experience Cloud (Target의 대상 소스)
* Adobe Experience Manager
* Target과 관련된 통계에서 사용되는 용어에 대한 기본 지식
* 쿠키 동작에 대한 기본 지식
* 솔루션 디자인 참조를 읽는 방법

환경:

* 데스크탑 및 모바일 장치를 포함한 데스크탑/웹 사이트
* 모바일 앱(기본 이해) / 모바일 장치용 Target 사용
* 스테이징 개발
* 프로덕션
* 모바일 장치, 태블릿 장치 및 데스크톱 장치에서의 프로덕션 환경
* 관련 호스팅 플랫폼
* 단일 페이지 애플리케이션에 target 사용
* 이메일

**시험 목표 및 범위**

섹션 1: 계획 및 전략 (31%)

* 비즈니스 KPI를 식별하고 주요 비즈니스 요구 사항을 고려하여 최적화 목표로 성공적으로 변환합니다
* 비즈니스 KPI 및 데이터 분석을 기반으로 최적화의 기회 파악
* 비즈니스 요구 사항, 노력 수준 및 상승도를 기반으로 테스트 아이디어 우선 순위 지정
* Adobe 샘플 크기 계산기를 기반으로 변수가 활동에 미치는 영향을 결정합니다
* 주어진 시나리오를 기반으로 테스트 가설을 만들고 가설 요소를 식별합니다
* 주어진 시나리오를 기반으로 테스트 가설을 분석하고 유효한 가설을 확인합니다
* KPI 기반 기본 및 보조 성공 지표 결정
* 대상 요구 사항 평가
* 적절한 분석 보고 소스(예: Target 또는 Adobe Analytics)을 결정합니다

제2절: 구성, 실행 및 관리(42%)

* 양식 기반 경험 작성기와 시각적 경험 작성기의 사용 설명
* VEC를 사용하여 경험을 만들려면 절차 적용
* 양식 기반 작성기를 사용하여 경험을 만들려면 절차 적용
* 경험 작성기 내에서 경험을 관리하기 위해 절차 적용
* Target 활동 만들기에 적용되는 워크플로우를 설명합니다
* 절차를 적용하여 A/B 테스트 활동을 만듭니다
* 절차를 적용하여 XT 활동 만들기
* 절차를 적용하여 MVT 활동 만들기
* 솔루션에 AP가 사용되는 방식을 이해합니다
* 절차를 적용하여 권장 사항 활동 만들기
* Target에서 대상을 만들기 위해 절차 적용
* 절차 적용을 통해 목표 및 설정 설정
* 활동 QA를 수행하기 위한 절차 적용
* Target 내에서 활동 및 대상을 관리하는 절차 적용

섹션 3: 분석 및 보고(17%)

* 보고서에 나타나는 요소를 설정하는 데 도움이 되도록 보고서 설정을 정의하고 평가합니다
* 평가 및 비교 [!DNL Adobe Target] 계획 Target 활동과 Target 활동 보고서의 유효성을 검사할 때 샘플 크기 계산기 설정
* A/B 테스트와 관련된 일반적인 문제와 이러한 문제를 방지하는 방법 평가 및 식별
* Automated Personalization 요약 보고서가 다른 보고서와 어떻게 다른지 이해합니다
* 경험 성과 보고서 평가(MVT)
* 관찰된 상승도, 신뢰도 수준 및 비즈니스 목표를 기반으로 채택 변형을 결정합니다

섹션 4: 문제 해결(10%)

* 활동 자격 문제 진단
* 브라우저 디버깅 도구(Experience Cloud Debugger 포함)를 사용하여 구현 확인
* 지표 및 보고 확인

**연습 테스트를 하세요**

시작하기 전에 무료 연습 시험을 보는 것이 좋습니다. Adobe의 전문가 수준 시험에 대해 사용 가능한 모든 실무 테스트에 대한 무제한 액세스가 제공됩니다.

우리의 연습 시험은 모두 실제 시험과 같은 청사진으로 개발되었다. 실습 테스트는 라이브 시험에서 다루는 주제를 이해하는 데 도움이 될 것입니다.

연습 시험에 합격한다고 해서 공인 시험에 합격할 수 있는 것은 아니라는 것을 명심하세요.

로 이동합니다. [Adobe 인증 시험 준비 포털](https://www.certmetrics.com/adobe/candidate/gmetrix_sso.aspx){target="_blank"} 지금 연습시험을 보기 위해.

**인증 시험 보기**

Adobe은 두 개의 공급업체를 사용하여 PSI와 Examity라는 가공 시험을 관리합니다.

* **PSI** 테스트 센터 및 일본어 시험 옵션 제공(사용 가능한 경우)
* **[!DNL Examity]** 원격 온라인 옵션 제공

시험을 예약, 다시 예약 또는 취소하려면 원하는 공급업체를 클릭하고 화면에 제공된 지침을 따르기만 하면 됩니다. 간단해!
